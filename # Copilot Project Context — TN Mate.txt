# Copilot Project Context — TN Materiales MVP

## 🎯 Project Goal

This project is a production-ready FastAPI application that integrates with Tiendanube (Nuvemshop) via OAuth.

The MVP includes:

- OAuth installation flow
- Store token persistence
- Product seed (GET /products paginated)
- Internal attribute storage (ancho_cm, composicion)
- CSV import/export
- Multi-tenant architecture (store_id required everywhere)

This is NOT a demo project. Code must be production-oriented.

---

# 🔐 OAuth Model

- OAuth flow: Authorization Code
- Tokens DO NOT expire
- store_id = user_id from token response
- We must persist:
  - store_id (PK)
  - access_token
  - status
  - installed_at

Never assume tokens expire.

Always send headers:

Authentication: bearer {access_token}
User-Agent: tn-attributes-mvp/1.0
Content-Type: application/json

---

# 🧠 Multi-Tenant Rule

Every table must include `store_id`.

Never write queries without filtering by store_id.

No global data allowed.

---

# 📦 Product Sync Rules

- Use paginated GET /products
- Use page + per_page
- Persist:
  - product_id
  - handle
  - title
  - updated_at (from TN)

Do NOT delete products automatically.
Future reconciliation may compare updated_at.

---

# 🧩 Attribute Model

We store attributes internally (not in Tiendanube yet).

attribute_definitions:
  - ancho_cm (number)
  - composicion (string)

product_attribute_values:
  - store_id
  - product_id
  - attribute_key
  - value

Unique constraint:
(store_id, product_id, attribute_key)

---

# 📂 CSV Rules

CSV import/export columns:

product_id, handle, title, ancho_cm, composicion

Import behavior:
- Skip unknown products
- Empty value = delete attribute
- Trim whitespace
- Never crash on malformed rows

---

# 🧱 Coding Standards

- Use async only for external IO (httpx)
- DB access stays sync (SQLAlchemy standard session)
- Always validate product existence before attribute write
- No business logic inside routes
- Use service layer for TN integration

---

# 🚫 What Copilot Must Avoid

- Do NOT hardcode store_id
- Do NOT assume single store
- Do NOT use global session
- Do NOT use deprecated SQLAlchemy syntax
- Do NOT mix sync + async DB engines
